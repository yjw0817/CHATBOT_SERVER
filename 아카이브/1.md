[Goal for Step 1]
- 로컬에서 바로 실행되는 웹앱을 만든다.
- SQLite DB 생성/마이그레이션 또는 자동생성
- 최소 UI 4개 화면 라우팅/탭 구성
- 문서 업로드(파일 저장 + 메타 저장 + 해시 계산 + 버전 처리)까지 동작

[Tech Choice]
- 네가 가장 빠르게 완성할 수 있는 스택 1개를 선택해.
  (권장: Python FastAPI + SQLite + Jinja/HTMX 또는 Node Express + SQLite + simple HTML)
- 프론트는 리액트 금지(시간 낭비). 단순 HTML/HTMX로.

[Must Implement]
1) DB tables:
apartments, documents, manual_sections, qa_issues, chunks,
conversations, messages, improve_suggestions, api_specs

2) Endpoints:
- GET / (home)
- GET /ui/upload
- GET /ui/quality
- GET /ui/chat
- GET /ui/improvements
- POST /api/apartments (create)
- GET /api/apartments
- POST /api/upload (multipart: apt_id, file)
- GET /api/docs?apt_id=

3) Upload logic:
- extract plain text from docx/pdf is NOT required yet (store file + placeholder raw_text).
- compute content_hash (sha256 of file bytes).
- if same apt + same hash exists: create new document row with version+1 and mark previous as ARCHIVED or keep latest pointer.
- store doc status = DRAFT.
