요청: “맥락 보강 비교(Compare Modal)”의 버튼 UX를 A안으로 변경해줘. (저장만 / 저장+Gate 분리)

[배경]
현재 Compare Modal 하단에 “적용 + Gate 재검증” 버튼만 있어 사용자가 저장만 하고 싶어도 Gate가 강제 실행됨.
RAG 품질 파이프라인은 Gate가 중요하지만, UX/비용/속도 때문에 “저장만”도 필요함.

[목표(A안)]
Compare Modal에 버튼 2개를 제공한다.
1) ✅ 적용(저장)  : Gate 실행 없이 해당 섹션 텍스트만 저장하고 모달을 닫는다.
2) ✅ 적용 + Gate 재검증 : 저장 후 즉시 해당 섹션에 대해 Gate#2를 실행하고, 배지/힌트/버튼 상태를 즉시 갱신한 뒤 모달을 닫는다.

[동작 정의]
- 공통: 오른쪽(After) 텍스트를 사용자가 수정할 수 있으며, “적용”은 오른쪽 텍스트를 section_text로 저장한다.
- 적용(저장):
  1) saveEditor() 또는 PUT /api/doc/{doc_id}/sections 로 section_text 저장
  2) Gate 호출은 하지 않는다
  3) UI에서 해당 섹션의 텍스트는 갱신하되, gate_status는 “미검증(UNVERIFIED)” 또는 기존 상태 유지 중 택1(아래 규칙)
  4) 모달 close

- 적용 + Gate 재검증:
  1) section_text 저장
  2) gateSection 호출하여 Gate#2 실행
  3) 반환된 gate_status/score/reasons로 섹션 배지/힌트/Fill 버튼 활성 상태 즉시 갱신
  4) 모달 close

[상태 처리 규칙(중요)]
- “적용(저장)”만 수행한 경우:
  - Publish(RAG 데이터화)는 이미 구현된 정책대로 gate_status='PASS'만 허용이므로,
    저장만 한 섹션은 바로 Publish에 포함되면 안 된다.
  - 따라서 아래 중 하나로 처리해줘(추천 1순위 포함):
    (추천) gate_status를 "NEED_FIX"로 강등하지 말고, 별도 플래그 `gate_stale=true`(또는 `needs_gate=true`)로 표시하여
           UI에 “⚠️ 변경됨: Gate 재검증 필요” 힌트를 띄우고 Publish/섹션 반영에서 제외되게 한다.
    (대안) gate_status를 "UNVERIFIED"로 설정하고 Publish에서 제외한다.
  - 문서/섹션 Gate 버튼을 누르면 이 플래그는 해제되고 최신 gate_status로 갱신된다.

[UI 요구사항]
- 버튼 라벨:
  - “적용(저장)”
  - “적용 + Gate 재검증”
- “적용(저장)”은 기본(primary)로 두어도 되지만, 혼동을 줄이려면
  “적용 + Gate 재검증”을 primary로 유지하고 “적용(저장)”은 secondary로 두는 것도 가능.
- 저장만 한 경우 해당 섹션에 힌트 표시:
  - “⚠️ 변경됨: Gate 재검증 필요”
  - Fill 버튼/Publish 버튼은 기존 Gate 상태를 신뢰하지 않도록(즉, 재검증 유도)

[구현 포인트(현재 흐름 반영)]
- 현재: applyCompare → saveEditor → gateSection (항상 Gate 실행)
- 변경:
  - applyCompareSaveOnly: saveEditor만 호출
  - applyCompareSaveAndGate: saveEditor 후 gateSection 호출

[테스트]
1) 저장만: section_text가 DB에 저장되고, gateSection 호출이 발생하지 않음
2) 저장+Gate: 저장 후 gateSection이 호출되고, gate_status가 즉시 갱신됨
3) 저장만 한 섹션은 Publish(reindex)에서 제외됨(또는 gate_stale=true로 제외)
4) 문서 Gate 또는 섹션 Gate 실행 시 stale 플래그 해제 및 최신 상태 반영

위 요구사항으로 Compare Modal UI와 로직을 변경해줘.
