요구: allow_qa 타입 혼선(boolean/string)으로 Q&A 정책이 뒤집히지 않게 “서버에서 표준화”하고 “프롬프트에는 정책 문장”을 조건 주입 방식으로 바꿔줘. (A안)

1) 서버에서 allow_qa 표준화(toBoolAllowQa)
- allow_qa 입력이 true/false(boolean) 또는 "true"/"false"(string) 등으로 와도 동일 동작하도록 캐스팅 함수 구현
- 기본값은 false (fail-safe)

예:
function toBoolAllowQa(v) {
  if (v === true) return true;
  if (v === false) return false;
  if (typeof v === "string") {
    const s = v.trim().toLowerCase();
    if (s === "true" || s === "1" || s === "yes") return true;
    if (s === "false" || s === "0" || s === "no" || s === "") return false;
  }
  return false; // default
}

2) 프롬프트에는 allow_qa 값을 그대로 넣지 말고, qa_policy_text를 조건적으로 주입
- allowQa=false일 때:
  qa_policy_text = "Q&A는 새로 추가하지 마세요. 기존 Q&A만 유지/정리하세요."
- allowQa=true일 때:
  qa_policy_text = "원문 근거가 명확한 경우에만 Q&A를 1~3개 추가할 수 있습니다. 답이 불명확하면 Q&A를 만들지 말고 [확인 필요]로 처리하세요."

3) FILL_SECTION_TEXT_PROMPT_V3 템플릿 수정
- {allow_qa} 대신 {qa_policy_text}를 포함시키고, 규칙 6번(Q&A 정책)은 qa_policy_text 내용을 따르게 구성

4) 최소 테스트 3개 추가(또는 수동 테스트 절차)
- allow_qa 미전송 → Q&A 신규 추가 절대 금지
- allow_qa: "true"(string) → 근거 있을 때만 Q&A 1~3개 추가
- allow_qa: true(boolean) → 위와 동일
