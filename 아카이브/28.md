[요청] Manualize 파이프라인을 “기존 방식(단일 호출 우선)”으로 롤백하세요.

[목표]
- Split/Chunk/Window 기반 분할 로직을 기본에서 사용하지 않고,
- 문서 전체(raw_text)를 1회 MANUALIZE_PROMPT_V3_1에 넣어 Manualize 수행합니다.
- 최종 품질(구조 보존/누락 최소화/환각 최소화)을 우선합니다.

[롤백 범위]
1) Split 단계 비활성화(삭제 또는 feature flag OFF)
- 헤더/번호/문단 기반 chunk 생성 OFF
- 글자수 window_size/overlap으로 여러 번 호출하는 방식 OFF (기본값)

2) Manualize 호출 방식 롤백(핵심)
- raw_text(문서 전체)를 그대로 MANUALIZE_PROMPT_V3_1에 “1회” 전달
- MANUALIZE_PROMPT_V3_1 내용은 변경 금지(오염 방지)
- 결과 저장은 기존 방식 유지

3) Gate/Fill/Merge는 현재 동작 그대로 유지
- Gate/FIll/Publish 로직 유지
- Split 전용 메타데이터(chunk_id 등)는 사용하지 않거나 무시

[예외 정책(최소화)]
- 원칙: 1회 호출 우선.
- 예외: 모델 입력 한계로 1회 호출이 실패할 때에만 “최소 분할(window)” 허용.
  · soft_limit(권장): 10,000자 — 1회 시도 우선, 실패 시에만 분할 검토
  · hard_limit(권장): 15,000자 — 1회 호출이 불가능하면 분할 강제

[검증 체크리스트]
- 문서 1개 업로드 → Manualize 호출이 원칙적으로 1회인지 로그 확인
- 분할/윈도우/반복 호출이 기본적으로 발생하지 않는지 확인
- 결과 섹션 구조가 원문 헤딩/번호 구조를 따르는지 확인
- Gate 실행/표시 정상 동작 확인
